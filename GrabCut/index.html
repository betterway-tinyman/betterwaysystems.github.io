GrabCut 컴파일 가이드

1. Dependencies

GrabCut 을 컴파일하기 위해서는, 아래 라이브러리들이 필요하다.

  • Boost
  • ITK (Insight Tool Kit)
  • Eigen

위 라이브러리드을 D:/Dependencies 폴더에 설치키로 한다. 또한, 각각의 컴파일 과정은 매우 오래 걸리니, 각 라이브러리 설치를 동시에 진행하는 게 좋다.

1.1. Boost 설치하기

http://www.boost.org/ 에 접속하여 최신 버전을 다운로드 받아 압축을 해제한다.

  • https://sourceforge.net/projects/boost/files/boost/1.62.0/boost_1_62_0.zip/download
  • 본인은 D:\Dependencies\Boost 에 압축을 풀었다.

CMD를 실행하여 다음 커맨드를 입력한다.

# Boost 가 설치된 경로로 이동
D:
cd D:\Dependencies\Boost

# 컴파일
bootstrap
b2 variant=debug,release link=static threading=multi address-model=64 stage

1.2. Eigen 설치하기

Eigen 은 헤더파일만 존재하니, 따로 컴파일할 필요는 없다.

https://bitbucket.org/eigen/eigen/ 에 접속하여 소스파일을 다운로드 받은 후, 압축을 풀어주기만 하면 된다.

  • https://bitbucket.org/eigen/eigen/downloads
  • D:\Dependencies\Eigen 에 압축을 풀었다.

1.3. ITK 설치하기

1.3.1. 프로젝트 구성하기

https://itk.org/ 에 접속하여 소스파일을 다운로드 받아 압축을 해제한다.

  • https://sourceforge.net/projects/itk/files/itk/4.10/InsightToolkit-4.10.1.zip/download
  • D:\Dependencies\ITK\Source 에 압축을 풀었다.

여기서부터는 CMake 프로그램이 필요하다. https://cmake.org/download/ 에 접속하여 프로그램을 다운로드 받은 후, 설치한다.

  • https://cmake.org/files/v3.7/cmake-3.7.0-rc2-win64-x64.msi

CMake 프로그램을 실행한 후, ITK 소스파일이 설치된 경로 및 빌드파일이 위치할 경로를 지정해준 후, Generate 버튼을 누른다. Generate 버튼 클릭 후, D:\Dependencies\ITK\Binary 폴더가 존재하지 않으니, 폴더를 만들겠다는 메시지가 뜬다. YES 를 눌러준다. 이후에는 VS 버전 및 아키텍처를 물어보는데, Visual Studio 14 2015 Win64 를 선택한다.

  1. Where is the source code: D:\Dependencies\ITK\source
  2. Where to build the binaries: D:\Dependencies\ITK\binary
  3. Generate 버튼 클릭
  4. Yes 버튼 클릭
  5. Visual Studio 14 2015 Win64 선택

1.3.2. 컴파일하기

binary\ITK.sln 를 실행하여, 빌드 타입을 설정한 후, 상단 메뉴의 Build -> Build Solution 을 클릭하여 컴파일한다. 여기서 선택한 빌드 타입과, 추후 GrabCut 에서 사용할 빌드 타입이 일치해야 한다.

  • GrabCut 를 Release 모드로 빌드하기 위해선, Release 모드로 빌드된 ITK 가 필요하다.
  • GrabCut 이 Debug 모드인데, Debug 모드로 빌드된 ITK 가 없다면, 컴파일 에러가 발생한다.

  1. 빌드 타입 선택
    • 본인은 Release 만 빌드하였다.
  2. Build -> Build Solution (또는 F7)

2. GrabCut 설치하기

2.1. 프로젝트 구성하기

다음 링크에서 소스를 다운받은 후, 압축을 푼다.

  • http://betterwaysystems.github.io/GrabCut/source.zip
  • 본인은 D:\OneDrive\Project\HisWill\GrabCut\source 에 압축을 품

CMake 를 실행한 후, source 와 binary 경로를 설정하고, Add entry 버튼을 눌러 다음 항목들을 추가해준다.

이름 설명 본인 설정값
EIGEN3_INCLUDE_DIR Eigen 경로 D:/Dependencies/Eigen
ITK_DIR ITK Binary 경로 D:/Dependencies/ITK/binary
Boost_INCLUDE_DIR Boost 헤더파일 경로 D:/Dependencies/Boost

그 후, Generate 버튼을 누르고, Visual Studio 14 2015 Win64 로 프로젝트를 구성한다.

2.2. 컴파일하기

binary\GrabCut.sln 을 실행한 후, 빌드 타입을 설정한 후, 상단 메뉴의 Build -> Build Selection 을 선택한다. 단, 이 때 선택된 빌드 타입으로 미리 컴파일된 ITK 라이브러리가 필요하다.

  • GrabCut 를 Release 모드로 빌드하기 위해선, Release 모드로 빌드된 ITK 가 필요하다.
  • GrabCut 이 Debug 모드인데, Debug 모드로 빌드된 ITK 가 없다면, 컴파일 에러가 발생한다.

2.3. GrabCut 실행하기

컴파일이 완료되었다면, binary\Release 폴더에 GrabCutExample.exe 가 존재할 것이다. 커맨드 라인에서 다음과 같이 입력한다.

  • GrabCutExample 타깃파일 마스트파일 출력파일
D:
cd D:\OneDrive\Project\HisWill\GrabCut\binary\Release

copy ..\..\source\data\soldier.png .
copy ..\..\source\data\soldier_selection.fbmask .
copy ..\..\source\data\soldier_selection.png .
