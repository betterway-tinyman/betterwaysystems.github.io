Class MultiSet<T>

An abstract set.

SetContainers are containers that store elements allowing fast retrieval of individual elements based on their value.

In an SetContainer, the value of an element is at the same time its key, used to identify it. Keys are immutable, therefore, the elements in an SetContainer cannot be modified once in the container - they can be inserted and removed, though.

SetContainer stores elements, keeps sequence and enables indexing by inserting elements into a List and registering iterators of the list container to an index table like {@link RBTree tree} or hash-table.

Container properties

Associative
Elements in associative containers are referenced by their key and not by their absolute position in the container.
Set
The value of an element is also the key used to identify it.
Multiple equivalent keys
Multiple elements in the container can have equivalent keys.

  • Inherited from SetContainer.handle_erase

    Abstract method handling deletions for indexing.

    This method, handle_insert is designed to unregister the first to last to somewhere storing those iterators for indexing, fast accessment and retrievalance.

    When erase is called with first to last, iterators positioning somewhere place to be deleted, is memorized and shifted to this method handle_erase after the deletion process is terminated.

    If the derived one is {@link RBTree tree-based} like TreeSet, the iterators will be unregistered from the tree as a tree node item. Else if the derived one is hash-based like HashSet, the first to last will be unregistered from the hash bucket.

    Parameters

    • first: SetIterator<T>

      An SetIterator to the initial position in a sequence.

    • last: SetIterator<T>

      An SetIterator to the final position in a sequence. The range used is [first, last), which contains all the elements between first and last, including the element pointed by first but not the element pointed by last.

    Returns void

    This method, handle_insert is designed to register the first to last to somewhere storing those iterators for indexing, fast accessment and retrievalance.

    Exchanges the content of the container by the content of obj, which is another container object with same type of elements. Sizes and container type may differ.

    After the call to this member function, the elements in this container are those which were in obj before the call, and the elements of obj are those which were in this. All iterators, references and pointers remain valid for the swapped objects.
