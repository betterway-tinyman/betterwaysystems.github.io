Protected performance

performance: number

Inherited from ParallelSystem.performance

  • Defined in typings/samchon-framework/samchon-framework.d.ts:5099

Performance index.

A performance index that indicates how much fast the connected parallel system is.

If this parallel system hasn't any Invoke message had handled, then the performance index will be 1, which means default and average value between all ParallelSystem instances (belonged to a same ParallelSystemArray object).

You can specify this performance by yourself, but notice that, if the performance index is higher then other ParallelSystem objects, then this parallel system will ordered to handle more processes than other ParallelSystem objects. Otherwise, the performance index) is lower than others, of course, less processes will be delivered.

This {@link performance index is always re-calculated whenever ParallelSystemArray calls one of them below.

  • ParallelSystemArray.sendSegmentData()
  • ParallelSystemArray.sendPieceData()

If this class is a type of DistributedSystem, a derived class from the ParallelSystem, then DistributedSystem.sendData() also cause the re-calculation.

    Checks whether the EventDispatcher object has any listeners registered for a specific type of event. This allows you to determine where an EventDispatcher object has altered handling of an event type in the event flow hierarchy. To determine whether a specific event type actually triggers an event listener, use willTrigger().

    The difference between hasEventListener() and willTrigger() is that hasEventListener() examines only the object to which it belongs, whereas willTrigger() examines the entire event flow for the event specified by the type parameter.

  • Implementation of IEntityCollection.refresh

    Dispatch a CollectionEvent with refresh typed.

    ICollection dispatches CollectionEvent typed insert or erase whenever elements I/O has occured. However, unlike those elements I/O events, content change in element level can't be detected. There's no way to detect those events automatically by IContainer.

    If you want to dispatch those typed events (notifying change on contents in element level), you've to dispatch refresh typed event manually, by yourself. Call refresh() with specified iterators who're pointing the elements whose content have changed. Then a CollectionEvent with refresh typed will be dispatched.

    If you don't specify any iterator, then the range of the refresh event will be all elements in this collection; {@link begin begin()} to {@link end end()}.

    Returns void
