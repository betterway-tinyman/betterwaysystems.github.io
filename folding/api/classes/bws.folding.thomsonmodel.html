Class ThomsonModel

톰슨 모형.

  1. 작업비
    • 고정비용: 30,000 원
    • 추가비용: 1,000 매 이후로 50 원 / 매
  2. 한 판 칼비 (도장, 판 only)
    • 5,000 원 / m
    • 추후 복잡도에 따라 가격이 변할 수 있다.

비고

  • 여유 용지: 면당 30매
  • 톰슨은 컷팅 행위를 행하기에 양/단면 구분이 없다. 오로지 단면만 존재함.

compute_spare_area

  • compute_spare_area(folder: Folder): number

  • Inherited from OptionalModel.compute_spare_area

    • Defined in bws/folding/price/OptionalModel.ts:70

    현 공정을 적용함에 있어 필요한 여분지, 여분지의 총 면적을 계산한다.

    각 공정마다 조판면을 구성함에 있어 각기 필요한 여분지 매수가 있다. 이렇게 필요한 모든 여분지의 전체 면적을 계산한다. 보통은 각 조판면의 면적에 대하여 여분지 매수를 곱하고, 이들을 합하여 총 면적을 구한다. OptionalModel.compute_spare_area 에서는, 이 보통의 경우를 계산한다.

    • 라미네이팅 코팅은, 각 조판면마다 여분용지가 필요한 게 아니고, 사용케 되는 용지 크기에 따라 여분 용지를 필요로 하게 된다. 때문에, LaminatingModel 은 별도의 여분용지면적 계산 메소드가 따로이 정의되어있다.

    Parameters

    • folder: Folder

      접지행위자

    Returns number

    필요 여분지에 대한 총 면적

    Get a XML object represents the Entity.

    A member variable (not object, but atomic value like number, string or date) is categorized as a property within the framework of entity side. Thus, when overriding a toXML() method and archiving member variables to an XML object to return, puts each variable to be a property belongs to only a XML object.

    Don't archive the member variable of atomic value to XML::value causing enormouse creation of XML objects to number of member variables. An Entity must be represented by only a XML instance (tag).
