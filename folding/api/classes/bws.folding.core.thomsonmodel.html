Class ThomsonModel

톰슨 모형.

  1. 작업비
    • 고정비용: 30,000 원
    • 추가비용: 1,000 매 이후로 50 원 / 매
  2. 한 판 칼비 (도장, 판 only)
    • 5,000 원 / m
    • 추후 복잡도에 따라 가격이 변할 수 있다.

비고

  • 여유 용지: 면당 30매
  • 톰슨은 컷팅 행위를 행하기에 양/단면 구분이 없다. 오로지 단면만 존재함.

Protected computeFaceCounts

  • computeFaceCounts(process: Process): Vector<Pair<Placement, number>>

  • Inherited from OptionalModel.computeFaceCounts

    • Defined in ts/src/bws/folding/core/price/OptionalModel.ts:105

    각 조판에 대한 판면수를 계산함.

    Placement 는 기본적으로 판면수를 가지고 있다. 하지만, 양면 인쇄를 행할 경우, 양단면 구분이 없는, 단면만을 가지는 현 모형의 특성상, Placement 가 가리키는 판면수를 그대로 사용할 수는 없다. (판면수를 절반으로 낮추어 사용해야함).

    computeFaceCounts() 는 이러한 경우에 대응키 위하여, 각 조판면별로 조정된 판면수를 산출해주는 함수이다. 현 공정은 홀수 판면에만 행해진다 가정하고, 이에 맞추어 조정된 판면수를 Placement와 묶은 std.Pair 객체들을 배열에 담아 리턴한다.

    Parameters

    • process: Process

      개별 공정 엔티티.

    Returns Vector<Pair<Placement, number>>

    Process에 속하는 각 조판 (Process) 및 해당 조판에 대한 판면수 Pair에 대한 Vector.

computeSpareArea

  • computeSpareArea(process: Process): number

  • Inherited from OptionalModel.computeSpareArea

    • Defined in ts/src/bws/folding/core/price/OptionalModel.ts:72

    현 공정을 적용함에 있어 필요한 여분지, 여분지의 총 면적을 계산한다.

    각 공정마다 조판면을 구성함에 있어 각기 필요한 여분지 매수가 있다. 이렇게 필요한 모든 여분지의 전체 면적을 계산한다. 보통은 각 조판면의 면적에 대하여 여분지 매수를 곱하고, 이들을 합하여 총 면적을 구한다. {@link OptionalModel.compute_spare_area} 에서는, 이 보통의 경우를 계산한다.

    • 라미네이팅 코팅은, 각 조판면마다 여분용지가 필요한 게 아니고, 사용케 되는 용지 크기에 따라 여분 용지를 필요로 하게 된다. 때문에, LaminatingModel 은 별도의 여분용지면적 계산 메소드가 따로이 정의되어있다.

    Parameters

    • process: Process

      접지행위자

    Returns number

    필요 여분지에 대한 총 면적

  • toXML(): XML

  • Implementation of IPriceModel.toXML

    Inherited from Entity.toXML

    • Defined in node_modules/samchon-framework/lib/ts/samchon-framework.d.ts:3294

    Get a XML object represents the Entity.

    A member variable (not object, but atomic value like number, string or date) is categorized as a property within the framework of entity side. Thus, when overriding a toXML() method and archiving member variables to an XML object to return, puts each variable to be a property belongs to only a XML object.

    Don't archive the member variable of atomic value to XML::value causing enormouse creation of XML objects to number of member variables. An Entity must be represented by only a XML instance (tag).

    Standard Usage.
