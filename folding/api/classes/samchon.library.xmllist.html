capacity

  • capacity(): number

  • Implementation of IArrayContainer.capacity

    Inherited from Deque.capacity

    • Defined in node_modules/@types/typescript-stl/index.d.ts:7436

    Return size of allocated storage capacity.

    Returns the size of the storage space currently allocated for the {@link IArray container}, expressed in terms of elements.

    This capacity is not necessarily equal to the {@link IArray container} size. It can be equal or greater, with the extra space allowing to accommodate for growth without the need to reallocate on each insertion.

    Notice that this capacity does not suppose a limit on the size of the {@link IArray container}. When this capacity is exhausted and more is needed, it is automatically expanded by the {@link IArray container} (reallocating it storage space). The theoretical limit on the size of a {@link IArray container} is given by member {@link max_size}.

    The capacity of a {@link IArray container} can be explicitly altered by calling member {@link IArray.reserve}.

    Returns number

    The size of the currently allocated storage capacity in the {@link IArray container}, measured in terms of the number elements it can hold.

  • Implementation of IArrayContainer.reserve

    Inherited from Deque.reserve

    • Defined in node_modules/@types/typescript-stl/index.d.ts:7404

    Request a change in capacity.

    Requests that the {@link IArray container} capacity be at least enough to contain n elements.

    If n is greater than the current {@link IArray container} capacity, the function causes the {@link IArray container} to reallocate its storage increasing its capacity to n (or greater).

    In all other cases, the function call does not cause a reallocation and the {@link IArray container} capacity is not affected.

    This function has no effect on the {@link IArray container} size and cannot alter its elements.

    Parameters

    • capacity: number

    Returns void

    Swap content.

    Exchanges the content of the container by the content of obj, which is another container object with same type of elements. Sizes and container type may differ.

    After the call to this member function, the elements in this container are those which were in obj before the call, and the elements of obj are those which were in this. All iterators, references and pointers remain valid for the swapped objects.

    Notice that a non-member function exists with the same name, swap, overloading that algorithm with an optimization that behaves like this member function.
