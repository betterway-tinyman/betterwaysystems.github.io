Class SharedWorkerServer

A SharedWorker server.

The {@link SharedWorker} is an abstract class is realized to open a SharedWorker server and accept web-browser clients. Extends this SharedWorkerServer class and overrides addClient() method to define what to do with newly connected remote clients.

Why SharedWorker be a server?

SharedWorker, it allows only an instance (process) to be created whether the SharedWorker is declared in a browser or multiple browsers. To integrate them, messages are being sent and received. Doesn't it seem like a relationship between a server and clients? Thus, Samchon Framework consider the SharedWorker as a server and browsers as clients.

The class SharedWorkerCommunicator is designed make such relationship. From now on, SharedWorker is a server and browsers are clients. Integrate the server and clients with this SharedWorkerCommunicator.

[Inherited] IServer

IServer is an interfaec for server classes who are providing methods for opening a server and accepting clients.

To open a server, extends one of derived class under below considedring which protocol to follow first. At next, overrides addClient() method who accepts a newly connected client as an IClientDriver object. Then at last, call open() method with specified port number.

Protocol Derived Type Related IClientDriver
Samchon Framework's own Server ClientDriver
Web-socket protocol WebServer WebClientDriver
SharedWorker SharedWorkerServer SharedWorkerClientDriver
