Interface IServer

An interface for a server.

IServer is an interfaec for server classes who are providing methods for opening a server and accepting clients.

To open a server, extends one of derived class under below considedring which protocol to follow first. At next, overrides addClient() method who accepts a newly connected client as an IClientDriver object. Then at last, call open() method with specified port number.

Protocol Derived Type Related IClientDriver
Samchon Framework's own Server ClientDriver
Web-socket protocol WebServer WebClientDriver
DedicatedWorker DedicatedWorkerServer DedicatedWorkerClientDriver
SharedWorker SharedWorkerServer SharedWorkerClientDriver

Below codes and classes will be good examples for comprehending how to open a server and handle remote clients.

  • https://github.com/samchon/framework/blob/master/ts/examples/calculator/calculator-server.ts
  • https://github.com/samchon/framework/blob/master/ts/examples/chat-server/server.ts
  • service.Server
  • external.ExternalClientArray
  • slave.SlaveServer

If you're embarrased because your class already extended another one, then use IServerBase.

    • Defined in node_modules/samchon-framework/lib/ts/samchon-framework.d.ts:5263

    Add a newly connected remote client.

    The addClient() is an abstract method being called when a remote client is newly connected with IClientDriver object who communicates with the remote system. Overrides this method and defines what to do with the driver, a newly connected remote client.
