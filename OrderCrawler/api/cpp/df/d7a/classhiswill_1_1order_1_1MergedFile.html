병합된 파일

OrderCrawler에 대하여 새로이 결정된 업무 규칙은 다음과 같다.

한 건의 주문 기록(Order)에 대하여, 단 하나의 상세정보(Detail)와 단 하나의 파일(File)만 보유할 수 있도록 한다. (본래 Order, Detail 및 File간의 집합관계는 1:M:N 이다). 만일, Order와 Detail 간의 관계가 1:1이 아니면 주문을 받지 않고, Detail과 File간 관계가 1:1이 아니라면, 해당 파일들을 하나로 병합하여 1:1 관계를 맞추도록 한다.

MergedFile는, 위 업무규칙을 따르기 위하여 정의된 클래스로써, Order와 Detail의 관계가 1:1이 아니거나, 혹은 Detail과 File간의 관계가 1:1이 아닌 1:N일 때, 단일 Order 내에 속한 모든 File 객체들을 하나로 병합함으로써 Order, Detail 및 File 객체간 집합관계를 1:1:1 관계로 만들어주기 위해 구현된 클래스다.

잘못된 코드:
이 업무규칙은 취소될 예정으로써, 그에 따라 MergedFile 엔티티도 없어질 것이다.
