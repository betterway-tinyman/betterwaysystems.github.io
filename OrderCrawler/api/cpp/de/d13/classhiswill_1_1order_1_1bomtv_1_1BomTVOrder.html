현재의 Order가 크롤링이 필요한 지 여부를 검사.

크롤링은 Contractor::loadPages 를 수행하여 개략적인 주문 기록 리스트를 생성하고, 세부적인 사항을 Order::load 를 호출함으로써 시행한다. needCrawling 은, 현재의 Order 가 load 가 필요한 지를 검사하는 메소드이다.

hiswill::order::Order를 구현.

auto BomTVOrder::toSurplusXML ( ) const -> std::shared_ptr<library::XML>
overrideprotectedvirtual

계약사 자체적으로 지니는 잉여정보를 XML로 리턴

공통(추상) 주문 엔터티와(Order)는 별도로, 각각의 계약사가 개별주문에 대하여 별도로 지니는 정보가 있다. 이러한 잉여(surplus)정보는 Contract 테이블의 surplus(XML) 필드에 삽입된다.

toSurplusXML() 메소드는 위 surplus 필드에 삽입될 XML 객체을 리턴한다.

반환값
잉여 정보를 대표하는 XML
